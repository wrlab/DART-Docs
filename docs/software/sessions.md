# Sessions & Logs

This page documents what DART saves and what to attach to bug reports.

---

## Session artifact (recommended)

At mission end, save:

- reconstruction output
- time-stamped trajectory
- synchronized video (optional)
- runtime metadata (versions, device models)
- logs

---

## Directory structure (suggested)

```text
sessions/
  2026-03-12_14-05-33/
    meta.json
    trajectory.csv
    reconstruction/
    video.mp4
    logs/
```

---

## Bug report checklist

Attach:
- server logs
- controller logs
- phone app logs (if available)
- a screenshot of the viewer
- your configuration values (IP/ports)
