# Contributing

DART is docs-first: improvements to documentation are valuable.

---

## Ways to contribute

- Improve build steps and checklists
- Add tested configurations (model numbers + photos)
- Add troubleshooting cases and fixes
- Improve scripts for reproducible installs

---

## Pull request guidelines

- Keep pages short and actionable
- Prefer checklists and “success criteria”
- Include screenshots/photos where helpful
